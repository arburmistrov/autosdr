Functional and technical documentation

E&R Legacy ERP Knowledge Base

The ERP supports order management, supplier management, inventory control, finance reconciliation, procurement approvals, customer records, and reporting. Documentation is structured so delivery teams and AI validation agents can understand system context before generating tests or release summaries.

Functional Requirements

  • Procurement managers can create and route supplier records.
  • Finance users validate payment details before export.
  • Approval owners can reject incomplete onboarding requests.
  • Audit logs must capture every supplier status transition.

Technical Notes

  • Monolithic ERP core with API adapter layer.
  • Nightly finance export remains fixed-width legacy format.
  • Inventory synchronization uses event-driven retry handling.
  • Role checks are implemented in the legacy permissions module.
